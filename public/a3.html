Fitness Tracker API Documentation

Welcome to the Fitness Tracker API documentation. This document provides details on how to use the various endpoints available in this RESTful service.

Base URL

The base URL for all endpoints is: http://localhost:2000/sets

Endpoints

1. Retrieve All Records

URL: /sets

Method: GET

Description: Retrieves all records from the set_details table.

Example Request:

GET http://localhost:2000/sets

Example Response:

[
  {
    "exercise_name": "Push-ups",
    "date": "2024-07-15",
    "sets": 2,
    "reps": 10,
    "weight": 100
  },
  ...
]

Error Responses:

{
  "Error": "Error in the SQL statement. Please check."
}

2. Retrieve Records by Exercise Name

URL: /sets/:exercise_name

Method: GET

Description: Retrieves records from the set_details table for the specified exercise name.

Example Request:

GET http://localhost:2000/sets/Push-ups

Example Response:

[
  {
    "exercise_name": "Push-ups",
    "date": "2024-07-15",
    "sets": 2,
    "reps": 10,
    "weight": 100
  },
  ...
]

Error Responses:

{
  "Error": "Error in the SQL statement. Please check."
}

3. Insert a New Record

URL: /sets/:exercise_name

Method: POST

Request Body:

{
  "exercise_name": "Push-ups",
  "date": "2024-07-15",
  "sets": 2,
  "reps": 10,
  "weight": 100
}

Example Request:

POST http://localhost:2000/sets/Push-ups
Content-Type: application/json

{
  "exercise_name": "Push-ups",
  "date": "2024-07-15",
  "sets": 2,
  "reps": 10,
  "weight": 100
}

Example Response:

{
  "Success": "Successful: Record was added!."
}

Error Responses:

{
  "Error": "Failed: Record was not added."
}

4. Update an Existing Record

URL: /sets/:exercise_name/:set_details_id

Method: PUT

Request Body:

{
  "exercise_name": "Push-ups",
  "date": "2024-07-15",
  "sets": 2,
  "reps": 10,
  "weight": 100
}

Example Request:

PUT http://localhost:2000/sets/Push-ups/1
Content-Type: application/json

{
  "exercise_name": "Push-ups",
  "date": "2024-07-15",
  "sets": 2,
  "reps": 10,
  "weight": 100
}

Example Response:

{
  "Success": "Successful: Record was updated!"
}

Error Responses:

{
  "Error": "Failed: Record was not updated."
}

5. Delete a Record

URL: /sets/:exercise_name/:set_details_id

Method: DELETE

Example Request:

DELETE http://localhost:2000/sets/Push-ups/1

Example Response:

{
  "Success": "Set details deleted successfully!"
}

Error Responses:

{
  "Error": "Set not found."
}

6. Retrieve a Record by Exercise Name and Set Details ID

URL: /sets/:exercise_name/:set_details_id

Method: GET

Example Request:

GET http://localhost:2000/sets/Push-ups/1

Example Response:

{
  "exercise_name": "Push-ups",
  "date": "2024-07-15",
  "sets": 2,
  "reps": 10,
  "weight": 100
}

Error Responses:

{
  "Error": "Set not found."
}

7. Find the Record with the Highest Weight for a Given Exercise

URL: /sets/:exercise_name/pr

Method: GET

Example Request:

GET http://localhost:2000/sets/Push-ups/pr

Example Response:

{
  "exercise_name": "Push-ups",
  "date": "2024-07-15",
  "sets": 2,
  "reps": 10,
  "weight": 150
}

Error Responses:

{
  "Error": "No record found for this exercise."
}

Input/Output Specifications

Input

For endpoints that require a request body (POST, PUT), the input should be in JSON format. Example:

{
  "exercise_name": "Push-ups",
  "date": "2024-07-15",
  "sets": 2,
  "reps": 10,
  "weight": 100
}

Output

The output for successful requests will be in JSON format. Example:

{
  "Success": "Successful: Record was added!."
}

For error responses, the output will be in JSON format. Example:

{
  "Error": "Error in the SQL statement. Please check."
}
